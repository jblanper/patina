Path A

The Annotation Drawer

A 340px contextual drawer, accessible from both the Ledger and the Cabinet. In the Ledger, triggered by a "Customize Display" text link placed between Edit and Delete in the header — the drawer opens from the left, overlapping the coin plate rather than the text record. In the Cabinet, triggered by a "Customize Display" link in the gallery header — the drawer opens from the right (left side is occupied by PatinaSidebar) and defaults to the Gallery Card tab. Toggle switches update both surfaces live — no Apply button, no navigation away.

✦Left anchor in Ledger: right folio (text record) stays fully clear
✦Live feedback: Ledger and Cards react instantly
✦Reachable from both surfaces — correct tab pre-selected
✦Toggle pattern matches existing sidebar design language
·Left drawer overlaps the coin plate in Ledger (acceptable trade-off)
·Right anchor in Cabinet overlaps the gallery cards while open

Path B

The Archivist's Manifest

A dedicated /preferences/display route, reachable by a "Display Preferences" text link in the Ledger header. The page presents two panels side-by-side: the Ledger (22 fields, grouped by section) on the left, and the Gallery Card (4 fields) on the right. Each field uses a compact ON/OFF text-toggle pair — typographically minimal, no slider chrome. Changes are batched and saved explicitly via "Save Preferences". A "Reset to Defaults" link is available at the top. Navigation back returns to the Ledger.

✦Full canvas for 22 fields — no overlay congestion
✦Card preferences share the same screen — no tab-switching
✦Accessible from Cabinet header too (global settings)
✦Batch save = single IPC call
·Navigates away from the Ledger (breaks Single-Click Rule for quick tweaks)
·No live preview — must return to Ledger to see effect

Path C

The Inline Edit Mode

A "Customize" button in the Ledger header activates a transient edit mode. In this mode, every field row in the Ledger gains a small visibility toggle button (rendered as a simple ○/● indicator to the right of each field). Toggling a field hides or shows it live, leaving a thin dashed stub — with a "Restore" link — where the hidden field was. A persistent mode-bar replaces the standard header, showing a live count ("12 of 18 fields visible") and a "Done" button. No panel, no route, no Apply. The Ledger itself becomes the settings UI.

✦Most contextual — you see the Ledger change as you configure it
✦No overlay, no navigation — zero context switch
✦No Apply button — simplest mental model
·Most complex to implement (edit mode adds state to CoinDetail)
·Card visibility has no natural home — needs a secondary interaction
·Dashed stubs add visual noise to the Ledger during edit mode
Interactive Demo — Click "Customize" to enter edit mode. Use ● toggles next to each field.
