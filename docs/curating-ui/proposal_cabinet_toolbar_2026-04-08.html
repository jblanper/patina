Cabinet Toolbar & Sidebar Toggle: Coherence & Clarity

The Ledger's top bar carries four distinct visual languages — ghost text link, icon toggle, outlined button, solid CTA — creating noise where there should be hierarchy. The sidebar collapse offers no affordance in its closed state: a lone chevron in a 44px strip reveals nothing of what lies behind it. This refinement unifies the toolbar into a three-zone grammar and restores legibility to the hidden-filter state.

  • Remove "CUSTOMIZE DISPLAY" as a standalone labelled text-link button
  • Absorb display customisation into the view-toggle group as a third icon button (columns icon)
  • Normalise TOOLS ▾ border to --border-hairline (resting state); Sienna only on hover/active
  • Three-zone toolbar: [display controls] / [tools] / [→ primary CTA]
  • Collapsed sidebar: add rotated "FILTERS" vertical label so the strip reads on first glance

Curatorial Audit

  • Visual Language

    Current: 4 distinct button treatments in one row. Proposed: 2 treatments — icon-group for display + outlined text for tools — with a clear primary CTA separated by auto-margin.

  • Affordance (Sidebar)

    A bare chevron fails the "zero context" test. The vertical "FILTERS" label costs 0 px of layout space and instantly communicates what is hidden. No hover required.

  • Interaction Feel

    The TOOLS button's Sienna border in its resting state competes with the primary CTA. Demoting it to hairline resting state preserves the accent colour for true active/hover feedback.

  • Touch Targets

    All icon buttons remain at ≥ 44px touch target. Vertical FILTERS label inherits full strip height click area — no accessibility regression.

Toolbar: What Changed & Why

  • "CUSTOMIZE DISPLAY" removed as a standalone text-link. The bare underline style was visually ambiguous — it looked more like a section label than an interactive button, and positioned left of the view toggle, it read as a descriptor for those icons rather than its own action.
  • A columns icon (three vertical bars) replaces it as the third button in the view-toggle group. Conceptually coherent: all three icons now govern "how you see the data." The action is still one click away.
  • TOOLS ▾ resting border drops to --border-hairline. Sienna is reserved for hover/active — the same contract already used by btn-view-mode. This removes the colour competition with + NEW ENTRY.
  • A hairline divider separates the icon group from TOOLS, establishing the three-zone grammar without adding visual weight.

Sidebar Toggle: What Changed & Why

  • In the collapsed state, a rotated "FILTERS" label runs the height of the strip below the chevron. Zero layout cost — it fits within the existing 44px width. No hover interaction required to understand the affordance.
  • The same writing-mode: vertical-rl + rotate(180deg) pattern is established CSS — no new primitives needed. Label inherits the button's existing mono/uppercase/muted-colour treatment.
  • The chevron itself flips to point right in the collapsed state (already implemented via rotate(180deg) on the SVG). Combined with "FILTERS", the strip now reads: "expand → to see filters."
  • The full strip remains the click target (44px × full height), satisfying the WCAG 2.5.5 touch target mandate without any change to the component's outer dimensions.
