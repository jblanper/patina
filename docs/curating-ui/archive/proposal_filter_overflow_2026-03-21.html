Archetype A

The Soft Reveal

The simplest intervention: show the top 8 values, fade the last entry into a gradient, and offer a clearly-counted inline link — "Show 9 more" — flush below. No new interaction patterns introduced. The collector who knows their vocabulary finds it immediately; the curious collector discovers the full breadth with a single click. Active selections are always pinned to the top of the list, never hidden.

Archetype B

The Indexed Archive

When a filter group exceeds 12 values, a search input appears above the list — narrow by typing a partial term, or browse the truncated list as usual. The metaphor is the card catalogue: the collector can browse the drawers, or go directly to a subject heading they know. Both modes coexist. Smaller groups (≤ 12) receive only the Show/Hide truncation from Path A, keeping the interface proportional to complexity.

Curatorial Audit — Path B

Contrast

Pass AA — Search input underline in --border-hairline. Focus state switches to --accent-manuscript.

Information Density

Dual mode: browse or search. Power users skip directly to "mint state" or "orichalcum" without scrolling any list.

Archival Tone

Good. The search input uses the underline-only form style from the existing design system, consistent with Scriptorium inputs.

Discovery Risk

Medium — search box may imply the visible list is exhaustive. Must pair with "Show N more" to clarify. Combined, discovery risk is low.

Active Selection Safety

Pass — search does not clear selections; active items retain state during and after filtering.

Implementation Cost

Medium — adds controlled input state per-group. Threshold logic (≥12 gets search) needs clear documentation.

Archetype C

The Curator's Ledger

Each filter group becomes a collapsible section — a closed drawer in the archive cabinet. A Burnt Sienna badge on the header counts active selections, so nothing is ever invisible. Within open groups, truncation + "Show N more" handles long lists as in Path A. The critical advantage: this architecture scales gracefully to future filter dimensions (denomination, mint, country, period) without the sidebar ever becoming unmanageable. Today it solves 4 groups; tomorrow it handles 10.

Curatorial Audit — Path C

Contrast

Pass AA — Badge: white on --accent-manuscript (#914E32): 4.8:1. Toggle arrow in --text-muted on --bg-manuscript: 3.2:1 (decorative, not text-sized).

Information Density

Highest structural density. Sidebar height is fully controlled by the user. Groups can be fully collapsed to labels-only, freeing space for many future filter dimensions.

Archival Tone

Strong. The collapsible group reads as a filing cabinet drawer — semantically correct for a collection management tool.

Discovery Risk

Low — Badges on collapsed groups communicate that selections exist. Groups open by default on first visit, so all dimensions are visible immediately.

Active Selection Safety

Pass — Badge always shows count. Active items pinned to top of truncated list. No selection is ever invisible.

Implementation Cost

Medium-High — Adds collapse state per group (persisted in localStorage for session continuity). Scalable architecture but more components to test.
