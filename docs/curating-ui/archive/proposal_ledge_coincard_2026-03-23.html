El Gabinete — Two Proposals

Two distinct UI problems identified in the Cabinet view. Each receives three curated paths. One path will be selected per problem and implemented together.

Problem I

The Ledge — Toolbar Redesign

The action bar above the coin grid currently presents five items at nearly identical visual weight: Personalizar Vista, Exportar Archivo, Generar Catálogo, Referencia de Campos, and + Nueva Entrada. The flat underlined treatment and tight horizontal packing produce visual noise. The hierarchy between a secondary utility action and the primary CTA is indistinguishable. Three restructuring strategies are proposed below.

Current State (Reference)
Path 1 The Command Strip

Secondary actions (export, catalogue, glossary) collapse into a single Herramientas ▾ dropdown. The toolbar is reduced to three items: a ghost Personalizar trigger, the tool menu, and the primary CTA. The dropdown surfaces on click — still within two interactions (Single-Click Rule compliant). Maximum reduction in visual noise.

Dropdown open ↓
↓ Exportar Archivo
☷ Generar Catálogo
☰ Referencia de Campos

Complexity: Low · Change scope: Cabinet.tsx + new dropdown component + CSS · Verdict: Cleanest surface, requires dropdown state management

Path 2 The Stratified Header Recommended

The toolbar splits into two tiers. The primary row holds only the object count and the CTA — clean, dominant, immediate. The secondary row presents utility actions as compact stone-pedestal chips with leading glyphs. Each tier has a clear purpose. No dropdown required. No hidden actions.

19 objetos en el registro.

Complexity: Very Low · Change scope: CSS only + minor Cabinet.tsx restructure · Verdict: Best hierarchy, no hidden state, fully responsive

Path 3 The Icon Ledge

All five actions remain in a single row but secondary tools become compact icon + micro-label vertical units. A hairline separator isolates the primary CTA on the right. The glyphs carry immediate meaning; the labels confirm it. Denser but more structured than current.

Complexity: Very Low · Change scope: CSS only · Verdict: Single-row constraint maintained; glyphs need careful selection for legibility

Problem II

CoinCard — Grade Display

The inline metrics row — ELECTRUM // 8.10G // 20.5MM // CHOICE XF — wraps mid-grade when the grade is multi-word, splitting a single token across lines. This breaks the rhythmic ledger aesthetic. Three solutions are proposed: one extracts the grade entirely, one gives it a dedicated row, and one resolves the overflow purely with CSS anchoring.

Current State (Reference) — Grade wraps on narrow cards

Path A The Grade Badge

Grade is extracted from the metrics row entirely. The metrics row becomes guaranteed single-line: Metal // Weight // Diameter. The grade surfaces as a solid accent badge, co-positioned with the catalog reference — asserting its importance as a collector-critical field without disrupting metric flow.

Grade badge sits in accent Burnt Sienna. Catalog ref and badge share one flex row — ref grows, badge stays pinned right.

Complexity: Low · Change scope: CoinCard.tsx restructure + CSS · Verdict: Grade gets its own visual identity; strongest hierarchy

Path B The Grade Row Recommended

A second metadata row is added below the metrics: Grado · VF-35. The metrics row remains a clean single line of physical data. The grade row is typographically subordinate but distinct — labeled, never orphaned. This mirrors professional numismatic catalogue layouts where grade is listed separately from physical specs.

Grade label in muted text-muted; grade value in Iron Gall at 600 weight — clear, never truncated, label disambiguates field meaning.

Complexity: Very Low · Change scope: CSS only (new .coin-grade-row) + trivial JSX change · Verdict: Cleanest to implement; aligns with numismatic convention

Path C The Anchored Grade

Minimal intervention. The metrics row gains flex-wrap: wrap and the grade span receives margin-left: auto. When the full row fits, grade floats to the right of the same line. When the card is narrow, grade drops to its own line left-aligned — never splitting mid-token. The stone-pedestal background on the grade provides subtle visual anchoring.

Curator's Recommendation

Preferred Combination

Ledge Toolbar

Path 2 — The Stratified Header

Splitting into two tiers requires zero new state management. The primary row achieves immediate hierarchy clarity — the count informs, the CTA acts. The secondary chip row respects the archival aesthetic without underlines or noise. Fully responsive. Requires only minor JSX restructure and new CSS chip styles.

Coin Card Grade

Path B — The Grade Row

A dedicated grade row aligns with professional numismatic catalogue conventions where physical specs (weight, diameter, metal) are grouped separately from condition (grade). The labeled approach removes ambiguity for new users. Implementation is CSS-only with a trivial JSX change — lowest risk, cleanest result.
