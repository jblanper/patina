# Technical Writing Style Guide

When drafting documentation, strictly adhere to the following rules:

## 1. Voice and Tone
- **Use Active Voice:** "The system processes the data" instead of "The data is processed by the system."
- **Present Tense:** Write in the present tense wherever possible.
- **Second Person:** Address the user directly as "you" (e.g., "You can configure..."). Use the imperative mood for instructions (e.g., "Click the button...").

## 2. Formatting (Docs-as-Code)
- **Semantic Headings:** Use ATX-style headings (`#`, `##`, `###`). Do not skip heading levels.
- **Code Blocks:** Always specify the language for syntax highlighting (e.g., ````python ```` or ````bash ````).
- **Inline Code:** Use backticks (` `) for file names, variable names, UI elements, and short commands.
- **Lists:** Use numbered lists (`1.`, `2.`) for sequential steps. Use bullet points (`-` or `*`) for non-sequential items.

## 3. Conciseness and Clarity
- **Be Direct:** Remove filler words (e.g., "simply", "just", "basically", "in order to").
- **Short Sentences:** Keep sentences under 25 words. Keep paragraphs under 4-5 sentences.
- **Progressive Disclosure:** Start with the most important information. Move deep-dive details further down or to separate explanation pages.
