The Scriptorium: Creation Interface Proposals

Objective: Design the "Add/Edit Coin" interface. It must balance the density of numismatic data with the "Manuscript Hybrid" aesthetic (breathable, prestigious, tactile).

Path A // The Ledger Slip
Modal Overlay
New Entry
Index Object
Philosophy: "Insertion." Acts like adding a slip to a physical file.
  • Pros: Maintains context (Gallery is visible behind). Fast interaction. Feels "lightweight."
  • Cons: Limited vertical space for rich data (Stories, multiple images). Can feel cramped on smaller screens.
  • Verdict: Best for quick entry, but risks crowding the "Curator's Note" or multi-image workflows.
Path B // The Curator's Desk
Dedicated Full Page
[LENS / IMAGE]
Athenian Owl
Permanently Archive
Philosophy: "Composition." A dedicated space for scholarly work.
  • Pros: Mirrors the CoinDetail layout (WYSIWYG). Maximal space for "Lens" preview and rich text stories. Distraction-free.
  • Cons: Requires navigation away from the Gallery. Heavier cognitive load.
  • Verdict: Most aligned with the "Prestige" and "Scholar" aesthetic. Encourages detailed cataloging.
Path C // The Field Journal
Slide-Over Sidebar
> QUICK INDEX
Save
Philosophy: "Field Note." Quick, transient data entry.
  • Pros: Very fast. Keeps Gallery fully visible. Good for "bulk" editing.
  • Cons: Extremely narrow. Bad for "Lens" image review. Feels like a "tool" rather than a "ritual."
  • Verdict: Efficient, but perhaps too utilitarian for the "Museum" vibe.

Curator's Recommendation: Path B (The Curator's Desk)

Rationale: The Patina philosophy prizes sanctuary and focus. Cataloging a historical object is a ritual. Path B provides the most respectful stage for this activity, allowing the user to view the high-resolution "Lens" image side-by-side with their data entry, mirroring the final CoinDetail presentation.

It also solves the space issue for the "Story" and "Provenance" fields, which require a rich writing environment.
