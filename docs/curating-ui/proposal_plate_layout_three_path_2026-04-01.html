Path A

Hero + Strip

The obverse commands the folio as a near-full-width hero frame. Reverse and Edge recede into a horizontal two-slot strip beneath — consistent with the visual grammar of major auction house catalogues (CNG, NAC) where the obverse is the identity face. The hierarchy is legible at a glance and survives the transition to a narrow sidebar.

  • Obverse holds commanding visual weight
  • Simultaneous view of all three faces
  • Natural reading order: primary → secondary
  • Strip pattern scales gracefully to narrow left folio
  • Strip thumbnails are small — detail only visible after capture
  • More vertical height than Path B

curating-ui Assessment — Path A

Numismatic convention

Obverse-first hierarchy exactly matches auction catalogue convention.

Simultaneous visibility

All three faces are visible at once during cataloguing.

Touch targets

Caption-area Replace / Remove buttons carry min-height: 44px — mandate satisfied.

Mobile fidelity

Strip collapses to a 2-col sub-row; primary retains 4:3 aspect — readable on small screens.

CSS complexity

Low — flex column + 2-col grid strip. No JS layout engine required.

Path B

Tabbed Plates

A single full-width frame, with Obverse · Reverse · Edge as a tab strip above. Only one plate is visible at a time. Clicking a tab switches the active slot. Eliminates vertical scroll entirely — the image area is maximised.

  • Maximum image area — full folio width
  • Zero vertical scroll
  • Cleanest visual — no competing frames
  • Reverse and Edge are hidden — no simultaneous comparison
  • Obscures missing-slot status at a glance
  • Tab pattern inconsistent with the rest of the Scriptorium UI

curating-ui Assessment — Path B

Simultaneous visibility

Fails. Collector cannot compare Obverse ↔ Reverse without tab-switching. This is a significant UX regression for cataloguing workflows.

Completeness feedback

Empty slots are invisible unless warning indicators are added to tabs — extra complexity.

Image area

Maximum — the full folio width is available for a single image.

Mobile

Clean — but no advantage over Path A on mobile since both stack to single-column.

Path C

Asymmetric Split

A 2-column sub-grid within the left folio: primary (Obverse) takes 63% of the width, and a right column holds Reverse stacked above Edge at 37%. All three plates are simultaneously visible. Conveys depth — the coin is shown from multiple angles at once.

  • All three faces visible simultaneously
  • Obverse dominates — 63% width is visually commanding
  • Compact vertical footprint
  • Secondary column is narrow — thumbnails may be too small on a 500px left folio
  • Replace / Remove actions crowded in narrow column
  • More CSS complexity than Path A

curating-ui Assessment — Path C

Obverse dominance

63% width is commanding but the left-folio itself is already only ~45% of the Scriptorium — the obverse ends up at ~28% of total window width. Less impactful than Path A.

Secondary column width

At 37% of ~500px folio = ~185px. Replace / Remove labels wrap or truncate. Touch targets borderline at that width.

Simultaneous visibility

Pass — all three faces visible at once.

CSS complexity

Higher — requires a nested 2-col sub-grid. Responsive collapse adds a media query at folio breakpoint (~600px), not only at viewport breakpoint.

curating-ui Verdict

Path A (Hero + Strip) is the preferred recommendation. It satisfies all five architectural constraints from the blueprint, matches numismatic publication convention, and keeps CSS complexity minimal. Path B is penalised by the simultaneous-visibility regression — hiding the reverse during cataloguing is a collector-experience step backwards. Path C is viable but the secondary column width creates touch-target pressure and adds CSS complexity without meaningful benefit over Path A.
