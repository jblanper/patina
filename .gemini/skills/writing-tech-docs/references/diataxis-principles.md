# Diátaxis Framework Principles

Diátaxis identifies four distinct user needs and maps them to four specific documentation types. **Never mix these types.**

## 1. Tutorials (Learning-Oriented)
- **Goal:** Allow the newcomer to learn by doing.
- **Focus:** The user's learning experience, not the product's features.
- **Rules:** Must be repeatable, build confidence, and result in immediate success. Do not explain *why* things work here; just show *how* to do it successfully.

## 2. How-To Guides (Problem-Oriented)
- **Goal:** Show a user how to solve a specific, real-world problem.
- **Focus:** Achieving a practical goal.
- **Rules:** Assume the user already has basic competence. Omit unnecessary explanations. Be highly structured with numbered steps. 

## 3. Reference (Information-Oriented)
- **Goal:** Describe the machinery and its operation accurately.
- **Focus:** The product/code itself.
- **Rules:** Must be austere, accurate, and complete. Structure around the codebase or UI (e.g., A-Z, by class, by endpoint). Do not include concepts or tutorials here.

## 4. Explanation (Understanding-Oriented)
- **Goal:** Clarify and illuminate a particular topic.
- **Focus:** Theoretical understanding, architecture, and historical context.
- **Rules:** Do not include instructions or step-by-step guides. Discuss alternatives, design decisions, and the "why" behind the software.
