Proposal: CoinDetail Responsive Refactor

Objective: Eliminate overflow issues and ensure flawless responsiveness across all viewports while maintaining the "Archival Ledger" aesthetic.

Path A: The Disciplined Ledger

Philosophy: Strict breakpoints with optimized spacing.

[ Mobile Stack ] -> [ Tablet Stack ] -> [ Desktop Split ]
  • Mechanism: Traditional Media Queries (`@media`).
  • Breakpoints:
    • < 900px: Single Column (Stacked)
    • > 900px: 45/55 Split
  • Overflow Control: `word-break: break-word` on all text. Rigid constraints ensure predictability.
  • Pros: Exact control over layout at specific sizes. Matches current logic but refined.
Path B: The Fluid Flex

Philosophy: Natural flow based on content size.

[ Flex Wrap: Items flow naturally ]
  • Mechanism: `display: flex; flex-wrap: wrap`.
  • Behavior: The "Right Folio" (Data) drops below the "Left Folio" (Image) whenever there isn't enough horizontal space to display both comfortably (e.g., < 800px).
  • Sizing: `flex-basis: 500px; flex-grow: 1`.
  • Pros: No hard breakpoints. extremely robust for odd window sizes.
Path C: The Modern Grid (Recommended)

Philosophy: "Intelligent" grid that adapts to available space.

[ Grid Auto-Fit ]
  • Mechanism: CSS Grid `repeat(auto-fit, minmax(min(100%, 450px), 1fr))`.
  • Behavior: Automatically switches from 2-column to 1-column when columns would be narrower than 450px.
  • Overflow Control: `min(100%, ...)` ensures columns never overflow mobile screens.
  • Pros: The cleanest code. Most robust "any size" support. Best alignment with modern CSS capabilities.
